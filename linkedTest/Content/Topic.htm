Delete Multiple Obsolete Nutrients

When a nutrient is inactivated in Formulation Cloud, it becomes obsolete for use within the client. The inactivated nutrient will be highlighted in yellow in the Master Nutrients window, and the check box in the Obsolete column will be selected.

For example, the following image shows that the <SWDIGPHOS nutrient is obsolete since it has a yellow background:

Obsolete nutrients should be deleted from all diets and products in the client, as the nutrient is no longer valid for use.

How to Delete Obsolete Nutrients:

  1. In the Master Nutrients window, identify obsolete nutrients by finding the ones that are highlighted in yellow and have the Obsolete check box selected.

    Click the Obsolete column heading to sort the nutrient list and group all obsolete nutrients together.

  2. Select the obsolete nutrients that you want to delete.

  3. Click the Delete button . The confirmation window appears.

    If the nutrients are being used in any product or diet, and the nutrients have a minimum or maximum constraint, the associated products and diets will be listed. This allows you to adjust the nutrients in the associated products or diets if needed before deleting the nutrients.

    If the nutrients are being used in any product or diet, but the nutrient does not have a minimum or maximum constraint, the associated products and diets will not be listed.

    For example, the following image shows that the obsolete nutrient, Niacin (B3), is used in multiple products and diets:

  4. Click the Delete button to continue the deleting process.

    Click Cancel if you want to make adjustments to the nutrient constraints in the associated products or diets before you delete the nutrients.

    The deleted nutrients will be removed from all diets and products in which they were used.

    The status of diets and products that contain obsolete nutrients that have either a minimum or maximum constraint changes to unformulated when the obsolete nutrient is deleted. The status of diets or products that contained the deleted nutrient that had no constraints does not change.

Only Current and Previous Database Backups Are Retained

When you install the program for the first time or upgrade the program to a new version, old backups of the database will automatically be deleted. This will conserve hard disk space as only two database backups – the current and previous – will be retained. Previously, all database backups were being retained.

New Ability to Delete Old Database Backups

When you back up your database, you can now delete old database backups.

How to Delete Old Database Backups
  1. Go to the System Setup > Utilities > System Data tab.
  2. Click the Backup Database button.
  3. Select the location to save the backup file.
  4. Click Save.

  5. After the backup has been saved, you will be prompted to open the backup location where you can manually delete old backup files that are no longer needed:

  6. In the Windows Explorer window, delete old backup files as needed.
  7. You should save your database backups to the same location each time for this process to be effective. If you don't follow this procedure, you are responsible for deleting your other outdated backups from other locations.

Get Customer Blend Enhancements

The Get Customer Blend process has been redesigned. You can now map/assign incoming ingredients to existing site ingredients or mark them as new before they are imported. This will prevent duplicate ingredients from appearing in the Site-Ingredients list after you get a blend from a different customer.

The Get Customer Blend window now appears within the main customer window as shown in the following image:

The new columns in the Ingredients grid are:

  • New – Select this option to import an incoming ingredient as a new ingredient for the current customer. If there are no matching ingredients, this option is automatically selected.

    Ingredients marked New appear in the Site-Ingredients list with the ingredient name followed by the blend name and customer name in parentheses. For example, Corn DDGS in the above image will appear as "Corn DDGS Test Blend Name(Test Customer)".

  • Target Ingredient – Use this drop-down menu to assign a blend ingredient to an existing site ingredient that has the same ingredient number and subcode. If there are no matching ingredients, this field is disabled and the New option will be selected.

Ingredient Name Color on Site-Ingredients Tab Not Turning from Blue to Black

In the Ingredients grid, the ingredient name will now turn from blue to black in the following scenarios:

  • On the Site-Ingredients tab, for an ingredient that has its name highlighted in a blue font and has nutrients that have a green and yellow background color, as you enter a value into the Amount AF or Amount DM column and tab out of the field for these nutrients, the background color of the nutrient will no longer be green or yellow. If you enter values into ALL the nutrients that are green and yellow, if you move off the ingredient and go to another one, the ingredient name will remain blue.
  • On the Site-Ingredients tab, after the AutoCalc program is run on an ingredient that has its name highlighted in a blue font and has some nutrients that have a green and yellow background color, those nutrients that were green and yellow and have had their values updated by AutoCalc in the Amount AF or Amount DM columns, the background color for these nutrients will no longer be green or yellow. If AutoCalc happened to update the values in all of the nutrients that were green and yellow, if you move off the ingredient and go to another one, the ingredient name will remain blue.
  • In the Quick Update window, for an ingredient that has its name highlighted in a blue font and has nutrients that have a green and yellow background color on the Site-Ingredients tab, after you enter a nutrient value for a highlighted nutrient and return to the Site-Ingredients tab, the background color of the nutrient will no longer be green or yellow. If you enter values into ALL the nutrients that are green and yellow for an ingredient, if you move off the ingredient and go to another one, the ingredient name will remain blue.
  • In the Lab Analysis window, after a lab analysis file is imported and an ingredient is selected that is blue on the Site-Ingredients tab and the Update button is clicked.
